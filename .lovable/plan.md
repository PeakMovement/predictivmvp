

# Streamlined Landing Page: Single Personalized Insight

## Overview

Transform the landing page from a static "system status" placeholder into a clean, focused experience that delivers **one highly personalized, actionable insight** the moment the user arrives.

## Current State vs. Desired State

| Current Index.tsx | Proposed Landing |
|-------------------|------------------|
| Static "System Status" text | Dynamic, personalized content |
| No user data integration | Pulls from Yves Intelligence engine |
| Generic welcome message | Context-aware greeting with user's name |
| No actionable information | Single clear "Today's Best Decision" |

## Design Philosophy

**"One thing that matters"** - Instead of overwhelming users with multiple collapsible sections, scores, and metrics, the landing page will:

1. Greet the user by name with their current context (injury status, stress level, goals)
2. Display ONE primary insight generated by the 4-layer reasoning engine
3. Show a clear, actionable session recommendation
4. Provide a single "dive deeper" path to the full Dashboard

## Data Flow

The landing page will use existing hooks:
- `useYvesIntelligence()` - Fetches the daily briefing and recommendations
- `usePersonalContext()` - Provides context chips
- `useTodaysDecision()` - Provides the structured session recommendation

No new edge functions needed - leverages the existing 4-layer reasoning engine.

### Greeting Logic

```typescript
function getTimeBasedGreeting(): string {
  const hour = new Date().getHours();
  if (hour < 12) return 'Good morning';
  if (hour < 17) return 'Good afternoon';
  return 'Good evening';
}
```

## What Gets Removed

From the landing experience:
- Focus Mode Selector (moved to Dashboard only)
- Layout Editor controls
- Multiple collapsible briefing sections (Brief of Day, Key Changes, Attention Needed)
- Diagnostics panel
- Score cards (Readiness, Sleep, Activity)
- Full recommendations list

These features remain accessible via the "View full briefing" link to Dashboard.

## User Flow

1. **User lands on `/`** → Sees personalized greeting + one insight
2. **Engagement options:**
   - "View full session" → Expands session details inline
   - "Log completed" → Marks session done
   - "See full briefing" → Navigates to Dashboard with all sections

## Handling Edge Cases

| Scenario | Behavior |
|----------|----------|
| No user logged in | Show login prompt (existing behavior) |
| Intelligence still loading | Show skeleton with pulsing animation |
| No insight available | Show "Generate your first briefing" CTA |
| AI credits exhausted (402) | Show friendly message to add credits |
| New user (no data) | Show onboarding prompt to connect Oura |
